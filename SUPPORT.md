# Support

Use GitHub issues for bugs, unclear behavior, documentation gaps, and feature requests.

Before opening an issue, include:

- the invocation you used
- what you expected
- what happened instead
- whether the request involved a URL, local app, screenshot, or codebase
- any relevant constraints such as framework, design system, accessibility, or performance requirements

Do not include private URLs, credentials, customer data, or confidential screenshots in public issues.
